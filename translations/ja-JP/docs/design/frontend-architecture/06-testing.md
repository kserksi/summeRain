# 06 · テスト戦略

> [!WARNING]
> **アーカイブ済みの設計記録です。** このページは完成済みの V2 フロントエンドより前の内容であり、
> バージョン、パス、実装状況が古くなっている可能性があります。

> 所属: [フロントエンド・アーキテクチャ設計（索引）](./README.md)

## 対象範囲

**Vitest + React Testing Library:**

- `lib/api.ts` を単体テストします（エンベロープの展開、CSRF の注入、**401 時のログアウト、4030 のアカウント無効化、429 のレート制限、エラーコードの i18n マッピング**）。
- MSW でエンドポイントをモックし、各機能の `hooks.ts` について Query/Mutation の動作を検証します。

アップロード、無限スクロール、フォーム検証、公開範囲の切り替え、パスワード変更後の再ログイン、登録後の遷移といった重要な操作にはコンポーネントテストを作成します。

## トレードオフ

網羅的なカバレッジは追求しません。システムの正しさの中核であり、回帰が生じやすい **API レイヤーと認証・Query フック** を優先します。

## 段階別の品質ゲート

- **コミット前（ローカルで高速）:** `prettier --check` + `eslint` + `tsc --noEmit`（[08](08-coding-standards.md)と整合）。
- **CI（マージとリリースを阻止）:** 上記 3 項目に **`vitest run` を追加**します。テストが失敗した場合はマージできず、リリースビルドでも同じテストスイートを実行します。
