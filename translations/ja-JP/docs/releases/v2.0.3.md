# v2.0.3 - フェイルファストなデータベースブートストラップ

> 開発プレリリースです。本リリースは `dev` から公開され、短期間で変更が
> 加えられる可能性があります。また、既定の `latest` イメージでは選択されません。

## 概要

v2.0.3 では、サーバー起動時に行うすべての MySQL スキーマ変更と設定変更を、
1 本の物理接続に紐づくアドバイザリーロックの下で直列化します。マイグレーション台帳の破損、
記録済みスキーマオブジェクトの欠落、旧式クリーンアップ、ベースラインマイグレーション、
バージョン付きマイグレーション、またはデフォルト設定の登録に失敗した場合、
サーバーはトラフィックを受け付ける前に起動を中止します。

本リリースでは、チェックサム付きスキーマバージョン、画像処理レシピ、アップロード API は
追加しません。既存のブートストラップ処理の調整、検証、およびエラー報告を変更します。

## 初回アップグレードの必須手順

最初の v2.0.3 インスタンスを起動する前に、v2.0.2 以下のすべてのバックエンドインスタンスを
停止してください。新旧バイナリーが重なるローリングデプロイは行わないでください。
旧バージョンは、v2.0.3 が現在所有するアドバイザリーロックの外側で、旧式クリーンアップ、
ベースライン `AutoMigrate`、およびデフォルト設定の登録を実行します。

1 つの v2.0.3 インスタンスがデータベースブートストラップを正常に完了した後は、
他の v2.0.3 インスタンスを通常どおり起動できます。同じデータベーススコープのロックによって
処理は直列化されます。

## 変更点

### 直列化された起動ブートストラップ

- 独立していた 4 つの起動呼び出しを、単一の `BootstrapDatabase(ctx, db)` に置き換えました。
- 1 本の物理 SQL 接続を占有し、その接続上で MySQL アドバイザリーロックを取得します。
  台帳検証、旧 token クリーンアップ、ベースライン、バージョン付きマイグレーション、
  デフォルト設定登録がすべて完了するまで、接続とロックを保持します。
- ベースラインと設定登録では、ロック済み物理接続を維持したまま新しい GORM session を使います。
- ロック待機とデータベース処理には呼び出し元の context を渡し、ロック解放には期限付きの
  クリーンアップ context を使用します。
- すべてのステージエラーを `main` へ返します。ブートストラップが部分的に失敗した後に
  HTTP サーバーを起動しません。

### マイグレーション整合性と復元

- 破壊的な旧式処理より前に、既知のバージョン、連続性、マイグレーション名、
  不変の SHA-256 チェックサムを検証します。
- 適用済み台帳項目に対応する列、インデックス、テーブル、および容量ロックのシード行が
  存在することを確認します。
- MySQL が一部の DDL をコミットした後、台帳記録前に停止しても、不足する処理を
  安全に再試行して収束できます。
- MySQL DDL が暗黙的にコミットされることを明記しました。復元は直列化、オブジェクト確認、
  冪等処理、再試行に基づき、コミット済み DDL のトランザクションロールバックを保証しません。

### 旧 token と設定の互換性

- 4 つの旧アクセストークン列を一度に確認し、実際に残っている旧列を 1 回の
  `ALTER TABLE` で削除します。
- `token_hash` が残っている場合だけ、互換性のない旧 token 行を削除します。
  hash のない部分移行済みテーブルでは、現行 token 行を保持し、残存する旧列だけを削除します。
- 既存の管理者設定を上書きしない競合安全な挿入を使い、照会または登録エラーを返します。
- 必要な場合、旧管理キー `image_token_default_ttl` を正式な
  `private_token_ttl_default_ms` へコピーし、既存の正式値は上書きしません。
- 管理 Web UI が、バックエンドと同じ正式なプライベート token TTL キーを読み書きするよう修正しました。

### 継続的インテグレーション

- バックエンド CI ジョブに固定バージョンの MySQL 8.4.10 サービスを追加しました。
- 実 MySQL ブートストラップスイートを必須化し、DSN がない場合はスキップせず CI を失敗させます。
- 1 接続のプール、反復実行、完全および部分的な旧 token、無効または不整合な台帳、
  context キャンセル、同時起動、一部 DDL コミット後の再試行、設定キー互換、seed エラー伝播、
  アドバイザリーロック解放を検証します。
- 依存関係のバージョンを変更せず、Go の直接依存関係宣言を整理しました。

## インストール

開発イメージは明示的に pull する必要があります。

```bash
docker pull jaykserks/summerain:dev-v2.0.3
```

同等の正確な開発タグ:

```text
jaykserks/summerain:dev-2.0.3
```

移動可能な `dev` タグも最新の成功した開発ビルドを指します。本リリースでは、
`latest`、`main`、安定版セマンティックバージョンのエイリアスは変更されません。

## 既知の制限

- 適用済みオブジェクト検証は、記録されたオブジェクトの存在を確認しますが、列型、
  デフォルト値、インデックス列順、制約、テーブルオプションを定義と完全比較しません。
- 本ブートストラップは MySQL DDL をトランザクションでロールバックできません。
  報告された問題を修正して再起動し、冪等な確認処理によってスキーマを収束させてください。
- アドバイザリーロックの待機は 60 秒です。履歴の大きなテーブルに対する DDL では、
  起動時間を十分に確保したメンテナンス時間帯が必要になる場合があります。
- 旧 `image_token_default_ttl` 行は、互換性のためコピー後も保持されます。実行時読み取りと
  新しい管理画面の書き込みは正式キーだけを使用します。
- アニメーション画像への対応は今後のリリースで予定されています。
