# v2.0.2 - 復元可能なブラウザーアップロードキュー

> 開発プレリリースです。本リリースは `dev` から公開され、短期間で変更が
> 加えられる可能性があります。また、既定の `latest` イメージでは選択されません。

## 概要

v2.0.2 では、ページの再読み込み、移動、タブ終了、一時的なネットワーク障害の後でも、
ブラウザーのアップロード作業を復元できます。ネットワーク処理を開始する前に、元ファイルと
V2 処理結果をユーザー別の IndexedDB キューへ保存し、アップロードページを再び開いたときに
サーバーのアップロードセッション状態と照合します。

本リリースには、データベースマイグレーションもサーバーアップロードプロトコルの変更もありません。

## 変更点

### 復元可能なブラウザーキュー

- ユーザーとキュー ID をキーとする IndexedDB キューを追加し、元ファイル、SHA-256
  フィンガープリント、処理レシピ、試行回数、公開範囲、セッション ID、再試行状態を保存します。
- ブラウザーの永続ストレージをベストエフォートで要求し、書き込み前に空き容量を確認します。
  少なくとも 32 MiB または全体の 10 パーセントを予備容量として残します。
- 永続化できない場合はアップロード全体を拒否せず、「このタブのみ」と明示した項目へ切り替えます。
- 24 時間の保持境界と遅延期限切れ削除を追加しました。完了または明示的に削除された項目は
  ストレージが利用できるときに直ちに削除します。
- サーバー処理開始後は保存済みの処理済み派生ファイルを削除し、元ファイルは完了、明示的な削除、
  または期限切れまでの範囲で保持します。

### V2 アップロードのクラッシュ復元

- 最初のパート PUT より前に、アップロードセッション ID と有効期限を確実に保存します。
- レシピが一致する場合、中断後も保存済み WebP パートを再利用し、元ファイルの再デコードと
  再エンコードを避けます。
- 再読み込み、画面移動、タブ終了、認証境界ではローカル作業だけを停止し、復元可能な
  サーバーセッションを暗黙にキャンセルしません。
- サーバー側のキャンセルは、ユーザーが項目を明示的に削除した場合だけ要求します。
- 冪等キーはキュー ID と正式な試行回数から生成され、中断した試行の二重公開を防ぎます。

### サーバー状態の照合

- 一括アップロード状態 API で稼働中の項目を照合し、混在したバッチが 4043 を返す場合は
  再帰的に分割して欠落 ID を特定します。
- 公開済み項目はローカルで完了し、有効なパートアップロードは再開し、サーバー処理中の
  セッションはポーリングします。以前のセッションが欠落または期限切れの場合だけ新しい
  冪等な試行を開始します。
- 状態照合が一時的に利用できない場合は、破壊的な変更を行わずキューを一時停止します。
- 中断した旧 V1 リクエストは結果不明かつ再試行不可として扱い、削除や再アップロードの前に
  「マイ画像」で確認するよう案内します。

### 複数タブとアカウント分離

- 更新可能な実行単位リースとトランザクション所有権チェックを追加し、永続項目を変更できる
  タブを同時に一つへ制限します。
- 手動再試行は保存済みの正式な試行回数とアップロード ID からアトミックに準備され、
  リース移譲後の古い画面状態による上書きを防ぎます。
- 認証世代ガードにより古いログインやプロフィール応答が以前のアカウントを復元することを防ぎます。
  ログイン、ログアウト、パスワード変更、別タブの認証イベントでは、現在のユーザー境界外の
  キューデータを削除します。
- 保存中、一時停止、復元可能、このタブのみ、ストレージエラー、V1 結果不明の表示を、
  英語、簡体字中国語、日本語で追加しました。

### 依存関係

- 実行時依存関係として正確なバージョン `idb` 8.0.3 を追加しました。
- テスト依存関係として正確なバージョン `fake-indexeddb` 6.2.5 を追加しました。

## 互換性とリソース動作

- IndexedDB、Web Crypto SHA-256、Blob、File API を使用し、Chromium 93 以降と
  導入済み iOS WebKit 605 系列を含む V2 ブラウザーマトリクスに対応します。
- ブラウザー画像処理は引き続き直列です。キューは二項目を並行調整し、一つのタブで有効な
  アップロードセッションを最大四つに制限して、復元や別タブ用の容量を残します。
- サーバー側画像処理、ストレージ配置、レシピ `2.0.0`、固定 WebP バリアントは変更されません。

## インストール

開発イメージは明示的に pull する必要があります。

```bash
docker pull jaykserks/summerain:dev-v2.0.2
```

同等の正確な開発タグ:

```text
jaykserks/summerain:dev-2.0.2
```

移動可能な `dev` タグも最新の成功した開発ビルドを指します。本リリースでは、
`latest`、`main`、安定版セマンティックバージョンのエイリアスは変更されません。

## 既知の制限

- 永続ストレージ権限がない場合、ブラウザーがキューデータを削除する可能性があります。
  最初の永続書き込みに失敗した項目は「このタブのみ」と表示されます。
- 24 時間の削除は遅延実行されるため、アップロードページを再び開くまで期限切れデータが残る場合があります。
- 旧 V1 アップロードには照会可能なセッション ID がなく、中断時は「マイ画像」での手動確認が必要です。
- アニメーション画像への対応は今後のリリースで予定されています。
- 対応する Chrome および iOS ブラウザーマトリクスでの実機受け入れ確認は、
  デプロイ時の検証手順であり、jsdom 単体テストには含まれません。
