# v2.0.1 - ブラウザー処理の事前検証

> 開発プレリリースです。本リリースは `dev` から公開されており、短期間で
> フォローアップ変更が加えられる可能性があります。また、既定の `latest`
> イメージでは選択されません。

## 概要

v2.0.1 では、負荷の高い画像デコードやアップロードセッションの作成前に、
V2 ブラウザー画像パイプラインがフェイルクローズするようになりました。
プロセッサーの選択では、ブラウザー機能だけに依存せず、wasm-vips と WebP
エンコーダーを実際にプローブします。また、画像処理を開始する前に、すべての
アップロードを完全なサーバーレシピと照合します。

本リリースには、データベースマイグレーションもサーバーアップロードプロトコルの
変更もありません。

## 変更点

### ブラウザー処理能力のネゴシエーション

- WASM ランタイム、HEIF/AVIF 動的ライブラリ、WebP エンコーダーを初期化する、
  共有かつ時間制限付きの wasm-vips worker プローブを追加しました。
- 初期化コストを再度支払わずに済むよう、プローブに成功した worker を最初の
  画像で再利用します。
- デコード後の寸法が端末固有の安全な Canvas 予算内にある場合に限り、
  ネイティブの pica/Canvas を使用します。
- wasm-vips が利用できない場合、大画像をアップロード前に拒否します。クライアントが
  暗黙のフォールバックとして元画像をバックエンドへ送信することはありません。
- 低メモリ端末を WASM 経路から除外する一方、`navigator.deviceMemory` がない
  ブラウザーは、実際の worker プローブによって対応を証明できます。

### 画像の事前検証

- 実際のコンテナー形式、拡張子と MIME の一致、アニメーション、寸法、元画像の
  50 MP 上限、ネゴシエーション済みのサーバーピクセル上限、WebP の一辺
  16,383 ピクセル上限を検証する、単一の事前検証プランを追加しました。
- 検査済みメタデータを処理時に再利用し、元画像コンテナーを二度解析しないように
  しました。
- 静止画の JPEG、JPG、PNG、BMP、WebP、AVIF 入力を引き続きサポートします。
- アニメーション PNG、WebP、AVIF、およびその他のアニメーション入力は、V2 では
  引き続きサポートされません。

### レシピ契約

- 数値上限と、必須の 4 バリアント `master`、`gallery`、`admin`、
  `publish_source` に対する厳密な Zod 検証を追加しました。
- 寸法、品質、フィットモード、パイプラインバージョン、レシピバージョンの変更を、
  処理前に安定したクライアントエラーで拒否します。
- no-store の API リクエストを使用し、拒否されたキャッシュエントリを保持せず、
  失敗したレシピ取得を無効化します。

### アップロードフィードバック

- 無効なファイル、未対応形式、アニメーション、寸法上限、プロセッサーの可用性、
  デコード/エンコード失敗、タイムアウト、未対応レシピ向けに、安定した
  `ClientImageError` コードを追加しました。
- 表示可能な `checking` 段階と、英語、簡体字中国語、日本語にローカライズされた
  行単位のエラーを追加しました。
- 無効なファイルは何も表示されずに消えるのではなく、明示的に拒否されたキュー行として
  保持されるようになりました。
- 再試行できない処理能力エラーと入力エラーでは、誤解を招く再試行操作を表示せず、
  一括選択時に大量の toast を発生させなくなりました。

## 互換性とリソース動作

- ネゴシエーション済みの最大 50 MP までの画像は、worker と WebP エンコーダーが
  実際のプローブに合格した後に限り、wasm-vips を使用します。
- Canvas フォールバックは、報告された端末メモリに応じて算出される 8～16 MP の
  安全予算内に引き続き制限されます。
- 画像処理は引き続き直列で実行され、各 worker はリクエスト後に終了するため、
  デコード済みピクセルと WASM メモリがキュー全体で蓄積することはありません。
- サーバーが受信するのは、レシピ `2.0.0` で定義された固定 WebP 派生画像のみです。

## 検証

- フロントエンド Vitest: 15 テストファイル、110 テスト成功。
- ESLint: 成功。
- TypeScript プロジェクトビルド: 成功。
- Vite 本番ビルド: 成功。
- 既存の wasm-vips 依存関係による direct-eval 警告は変更されておらず、ビルドを
  失敗させるものではありません。

## インストール

開発イメージは明示的に pull する必要があります。

```bash
docker pull jaykserks/summerain:dev-v2.0.1
```

同等の正確な開発タグ:

```text
jaykserks/summerain:dev-2.0.1
```

移動可能な `dev` タグも、最新の成功した開発ビルドを指します。本リリースでは、
`latest`、`main`、安定版セマンティックバージョンのエイリアスは変更されません。

## 既知の制限

- ブラウザーのアップロードキューはまだ永続化されていません。アップロードページを
  再読み込みする、閉じる、または別ページへ移動すると、現在のキューは中断されます。
- wasm-vips を初期化できない端末では、安全な Canvas 予算を超える画像を
  アップロードできません。
- アニメーション画像への対応は、今後のリリースで引き続き予定されています。
- 対応する Chrome および iOS ブラウザーマトリクスでの実機受け入れ確認は、
  デプロイ時の検証手順であり、jsdom 単体テストには含まれません。
