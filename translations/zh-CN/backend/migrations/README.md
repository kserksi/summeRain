# 数据库结构迁移

后端通过 `repository.ApplySchemaMigrations(db)` 管理仅增量的 MySQL 数据库结构
迁移。该函数一旦返回错误，启动过程必须立即终止。

迁移执行器会：

- 独占一个 SQL 连接，并持有数据库范围的 MySQL advisory lock；
- 按需创建 `schema_migrations`；
- 校验每个已应用版本不可变的 SHA-256 校验和；
- 按版本顺序应用缺失的操作；
- 在添加列或索引前查询 `information_schema`；
- 仅在某版本的全部操作成功后记录该版本。

MySQL DDL 会隐式提交。因此，每项操作都必须是增量操作，并能在中断后安全重试。
绝不能编辑已经应用的迁移；应追加更高版本。破坏性的重命名、删除列和删除表必须有
单独的兼容与回滚方案，因此 V2 引导迁移有意不包含这些操作。
