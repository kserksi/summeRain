# 06 · 测试策略

> [!WARNING]
> **已归档的设计记录。** 本页面早于已经完成的 V2 前端，
> 其中的版本、路径或实现状态可能已经过时。

> 所属：[前端架构设计（索引）](./README.md)

## 范围

**Vitest + React Testing Library：**

- 对 `lib/api.ts` 编写单元测试（信封拆解、CSRF 注入、**401 登出、4030 账号停用、429 限流及错误码国际化映射**）。
- 使用 MSW 模拟接口，测试各特性的 `hooks.ts`，验证查询与变更行为。

对关键交互编写组件测试：上传、无限滚动、表单校验、可见性切换、修改密码后的重新登录，以及注册成功后的跳转。

## 取舍

不追求全量覆盖。优先测试 **API 层与鉴权/查询钩子**，因为这两部分是系统正确性的核心，也最容易发生回归。

## 分层质量门禁

- **pre-commit（本地、快速）：** `prettier --check` + `eslint` + `tsc --noEmit`（与 [08](08-coding-standards.md) 保持一致）。
- **CI（阻断合并与发布）：** 上述三项检查**加上 `vitest run`**。测试失败时不得合并，发布构建同样运行该测试套件。
